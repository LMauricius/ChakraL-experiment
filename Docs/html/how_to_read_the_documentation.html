    Documentation structure

    The documentation is structured in several chapters: * Introduction - introduction to the language, its goals, and stuff needed to start writing the code * Basic concepts - helps understand the language and its terminology * Language features - describes all the features of the language in detail * Standard library - describes the modules included with the compiler * Reference compiler - how to use the reference compiler * Style guide - the recommended writing style for readability
